# Платформа для аренды жилья

Проект по проектированию и реализации базы данных для системы аренды жилья.
## ER-диаграмма

Первым делом я создал **ER-диаграмму** для проекта, чтобы четко определить структуру будущей базы данных и взаимосвязи между сущностями.

## Структура базы данных

Используя разработанную ER-диаграмму, я создал следующие таблицы в **PostgreSQL**:

### Основные таблицы:
| Таблица | Описание |
|---------|----------|
| `users` | Пользователи платформы |
| `owners` | Владельцы жилья |
| `housing` | Объекты жилья |
| `amenities` | Удобства |
| `housing_amenities` | Связь жилья и удобств |
| `statuses` | Статусы бронирований |
| `reservations` | Бронирования |
| `reviews` | Отзывы |

### Ключевые связи:
- **Владельцы** → **Жилье** (один ко многим)
- **Пользователи** → **Бронирования** (один ко многим)
- **Жилье** → **Бронирования** (один ко многим)
- **Жилье** → **Удобства** (многие ко многим)
- **Пользователи** → **Отзывы** (один ко многим)

## Заполнение базы данных

Для тестирования функционала я заполнил таблицы реалистичными данными:

- **105 пользователей** с российскими именами и контактами
- **50 владельцев** жилья  
- **75 объектов** жилья разных типов
- **110 бронирований** с различными статусами
- **105 отзывов** с рейтингами от 1 до 5

Все данные связаны между собой, что позволяет тестировать сложные SQL-запросы.
